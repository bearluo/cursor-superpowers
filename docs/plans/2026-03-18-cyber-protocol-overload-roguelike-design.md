# 赛博协议：过载与腐化（Roguelike 割草）设计稿

**日期**：2026-03-18  
**定位**：局外养成 + 局内双摇杆割草战斗（Roguelike/副本）  
**核心特色**：元素/状态“协议化” + 反应链构筑 + 腐化阈值风险 + 主动过载爆发  

---

## 1. 一句话卖点（Pitch）
玩家是“觉醒实验体”，能把城市异常能量编译为战斗协议；通过选择协议模块与触发顺序打出反应链清屏。战斗中腐化持续攀升，你可以主动“过载”换取爆发窗口，但会加速腐化并引发失控事件——在失控边缘用正确组合封神。

---

## 2. 目标体验与设计原则
- **爽点密度**：每 2–3 分钟让 Build 明显“变强一次”（新反应/更大范围/更高连锁）。
- **高可读性**：反应结果必须“看得懂、学得会、可复现”（避免黑箱数值堆叠）。
- **风险可预测**：腐化阈值与失控事件给出明确预期，玩家是在“押注”，不是被随机惩罚。
- **低资产起步**：先做机制闭环与节奏，再扩展内容与表现（适合迭代式开发）。

---

## 3. 核心循环（Core Loop）

### 3.1 局内（单局）
进入异常区 → 清怪/拾取经验与模块 → 选择协议/反应强化 → 触发精英/事件/商店节点 → 守门者战 → 继续深入或撤离结算。

### 3.2 局外（长线）
结算获得样本/零件/研究点 → 提升稳定度与抑制器（容错与阈值控制）→ 解锁协议研究树（玩法分支）→ 改造机体插槽（手感与流派偏好）→ 再次进入异常区。

---

## 4. 协议、状态与反应（MVP）

### 4.1 MVP 协议（4 个）
> 统一模型：协议对敌人施加对应 **Stacks**（0–100），当满足阈值/条件时触发反应事件。

- **热协议（Ignite）**：灼烧 DOT；高层可扩散。
- **冷协议（Freeze）**：霜化减速；满层冻结短控。
- **毒协议（Toxin）**：中毒 DOT；对高血量目标更有价值。
- **电协议（Arc）**：导电；触发连锁在目标间跳跃。

### 4.2 最小反应表（可玩且可读）
- **热 + 电：过热放电** → 小范围爆炸 + 连锁跳数 +1
- **冷 + 电：超导** → 大范围减速/易伤（控场聚怪核心）
- **冷 + 热：热震** → 清空两者 stack，造成一次高额爆发（高频爽点）
- **毒 + 热：燃毒** → DOT 变为可扩散形态（割草清屏核心之一）
- **毒 + 电：神经脉冲** → 连锁附带短暂麻痹（生存向）

### 4.3 反应可读性要求（实现约束）
- 每次触发反应必须有：**图标/颜色**、**范围提示**、**数值/状态浮字**三者至少其二。
- UI 需要能展示：当前携带协议、关键反应说明、腐化阈值与风险预告。

---

## 5. 腐化（Corruption）与过载（Overload）

### 5.1 腐化条（风险骨架）
- **上升来源**：时间推进（基础增长）+ 受击/濒死 + 触发高阶反应 + 使用过载。
- **抑制来源**：少量净化掉落（精英/事件）或防御型协议/模块。
- **阈值分段**：建议 3 段（例如 33/66/100），每段都应改变局内环境与风险预期。

### 5.2 主动过载（核心操作点）
- **启动**：一个主动键（冷却或消耗“过载电容”）。
- **窗口**：短持续（建议 6–10 秒）。
- **收益**：
  - 反应触发阈值降低或倍率提高（更容易连锁、更大范围/更高伤害）
  - 协议获得“过载词条”（如电跳数、毒扩散半径、冰冻结稳定度等）
- **代价**：
  - 腐化瞬间激增 + 过载期间腐化增长更快
  - 过载结束进入短暂“衰减”（如移速/攻速下降或承伤上升），形成押注窗口

### 5.3 失控阈值事件（把代价具象化）
当腐化跨过阈值（或过载结束且腐化过高）触发事件，MVP 三类：
- **追猎者降临**：刷追猎精英，击败掉落净化资源/高阶模块（高风险高回报）。
- **协议反噬**：随机封锁一个协议一段时间（迫使临场调整 Build）。
- **环境异常**：场地出现电弧/毒雾/寒潮等区域危害（提升走位压力）。

> 关键原则：失控不应是纯惩罚，它必须也是“拿资源/拿强度”的入口。

---

## 6. 关卡结构与单局节奏（10–15 分钟）

### 6.1 MVP 结构（推荐：分区波次制）
- 每区 **3–5 波**：小怪 → 精英 → 事件/商店 → 下一分区
- 每区末 **三选一奖励**：协议模块 / 净化 / 义体零件
- 每 2–3 区一个 **守门者**：要求玩家用反应链“破机制”，避免纯数值碾压

### 6.2 节奏目标（体验配方）
- **0–2 分钟**：建立 1–2 协议，读懂 stack 与反应提示
- **2–6 分钟**：反应链成型；引导第一次过载押注
- **6–10 分钟**：腐化逼近第二阈值；出现失控事件，体验风险换资源
- **10–15 分钟**：终局冲刺：高腐化 + 连续过载窗口 + 守门者机制，形成封神/翻车分水岭

---

## 7. 世界观（服务玩法的最小叙事）

### 7.1 世界设定
- **舞台**：被“协议风暴”撕裂的超巨型都市，现实层与异常层叠加，底层规则被协议化。
- **异常来源**：跨层实验泄露，使“协议”成为可传播的能量规则；你是实验体，能把协议内化并编排。

### 7.2 玩家身份与动机
- **身份**：觉醒实验体/异常共鸣者；越战斗越异化。
- **动机**：寻找“协议源点/主控密钥”，决定治愈、共生或改写城市规则（并摆脱追捕）。

### 7.3 组织三角（冲突结构）
- **公司**：试图武器化协议并回收实验体。
- **黑市/清道夫**：交易异常模块与改造技术。
- **封锁局/秩序派**：主张封城净化，与你的异化变强天然冲突。

---

## 8. 首发 MVP 内容清单（可落地）

### 8.1 玩法系统（必须）
- 4 协议（热/冷/毒/电）+ stack 模型
- 5 条反应（见 4.2）+ 可视化反馈
- 腐化条（分段阈值）+ 过载主动技能 + 衰减
- 3 类失控事件（追猎者/协议反噬/环境异常）
- 分区波次制关卡骨架 + 2 个守门者（至少一个强调控场、一个强调爆发）
- 奖励三选一 + 简易事件/商店（至少二选一）

### 8.2 局外（最小但可循环）
- 稳定度/抑制器：影响腐化阈值与容错
- 协议研究：解锁少量反应变体或过载词条
- 机体插槽：影响手感（移速、拾取范围、过载冷却等）

### 8.3 UI/引导（必须）
- 局内 HUD：协议列表、反应提示、腐化条/阈值预告、过载状态
- 简易教程：第一局引导“触发一次反应 + 按一次过载”

---

## 9. 可扩展方向（后续迭代）
- 扩展协议到 6–8 个（例如磁、蚀、光、暗），但坚持“反应可读性”原则。
- 增加“反应变体”而非纯数值（同一反应可分控场/爆发/扩散三系）。
- 引入形态化异化（从本稿方案 R 平滑进化到方案 S）：高腐化解锁形态/器官树。
- 增加叙事节点：失控事件触发分支，影响阵营态度与结局。

---

## 10. 需要进一步确认的开放点（实现前定稿）
- 单局时长与节奏是否固定 10–15 分钟，还是允许 20+ 分钟长局。
- 过载资源：冷却制 vs 充能制（击杀/反应充能）。
- 反应表的数值调性：更偏爆发清屏，还是控场生存。
