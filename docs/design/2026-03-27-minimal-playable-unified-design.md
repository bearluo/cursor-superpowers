# 最小可玩统一设计（Unified PRD）

**状态**：active  
**日期**：2026-03-27  
**整合来源**：`docs/design/features/feature-battle.md`、`docs/plans/2026-03-18-cyber-protocol-overload-roguelike-design.md`、`docs/plans/2026-03-19-playable-demo-design.md`  
**定位**：作为“最小可玩”唯一口径文档，用于后续实现与验收。

---

## 1. 目标与范围

### 1.1 目标体验（10-15 分钟单局）
- 玩家通过自动攻击叠加协议并触发反应。
- 玩家主动使用过载进行风险押注，换取短时强度窗口。
- 腐化跨阈值触发失控事件，形成可预期的风险压力。
- 刷怪压力与经验拾取共同驱动节奏，最终形成“变强-押注-冲刺”闭环。

### 1.2 并集硬性范围（必须满足）
- 4 协议：热/冷/毒/电；统一使用 Stack 模型（0-100）。
- 5 反应：过热放电、超导、热震、燃毒、神经脉冲。
- 腐化阈值：33/66/100；过载具备收益与代价。
- 玩家输入：`W/A/S/D` 移动，`Space` 过载，`1/2/3` 奖励选择。
- 至少 1 类失控事件具备可观测影响（其余可提示级）。

### 1.3 分层交付
- Demo 必达层：可玩闭环、可读反馈、可复现触发。
- 扩展层：守门者、更多失控事件实体化、更多局外内容。

### 1.4 非目标（本轮不强制）
- 复杂 AI（包围、预测拦截等）。
- 双摇杆/鼠标瞄准、多技能连招。
- 正式美术与复杂特效管线。

---

## 2. 架构与职责边界

### 2.1 分层职责
- `systems/*`：协议/反应、腐化/过载、波次与强度曲线、经验与奖励规则。
- `features/*`：输入、HUD、奖励界面、战斗表现编排。
- `core/*`：`EventBus`、常量、通用类型。

依赖方向必须满足：`features -> systems/framework -> core`。

### 2.2 波次与奖励职责（统一口径）
- **波次只负责**：刷怪频率、刷怪批次、怪物强度递增与场上压力。
- **奖励只负责**：经验球拾取达阈值后触发三选一。
- 禁止“清波即奖励”的直连逻辑。

### 2.3 关键事件契约（最小集）
- `REACTION_TRIGGERED`
- `OVERLOAD_STARTED` / `OVERLOAD_ENDED`
- `CORRUPTION_THRESHOLD_CROSSED`
- `MELTDOWN_EVENT_TRIGGERED`
- `XP_PICKED`
- `REWARD_READY` / `REWARD_SELECTED`

---

## 3. 核心玩法闭环与状态机

### 3.1 单帧主流程
输入采样 -> 自动攻击施加协议 -> 反应判定与结算 -> 腐化更新与阈值检查 -> 失控事件注入/提示 -> 刷怪调度 -> 经验拾取与奖励判定。

### 3.2 状态机
- 过载：`ready -> active -> cooldown -> ready`
- 腐化阶段：`phase1 -> phase2 -> phase3`
- 奖励：`collecting_xp -> reward_ready -> picking -> collecting_xp`

---

## 4. 数值与规则基线（MVP）

### 4.1 协议与反应
- Stack 范围：0-100。
- 反应触发基线：目标具备双协议且达到阈值（建议起点 20/20，可按联调调整）。
- 反应反馈：每次触发需满足“图标/范围提示/浮字”至少两项。

### 4.2 腐化与过载
- 腐化阈值：33/66/100。
- 过载窗口：建议 6-10 秒（默认 8 秒）。
- 过载收益：至少落地 1 项（阈值降低、反应倍率提升、叠层加速）。
- 过载代价：必须同时包含“启动瞬增 + 持续增速”。

### 4.3 经验球与奖励触发（强约束）
- 奖励等级：`rewardLevel` 从 1 开始。
- 阈值公式：`xpNeed(level) = round(8 + 3*level + 1.5*level^1.35)`。
- 触发规则：`xpCurrent >= xpNeed` 时进入奖励选择并扣除阈值。
- 溢出规则：保留超出经验，继续累计下一级奖励。

---

## 5. 可玩验收标准（并集最严格）

10 分钟内应稳定满足：
- 至少触发 3 种不同反应，且可见反馈充分。
- 至少触发 1 次过载并可感知强度提升。
- 腐化至少跨越 1 个阈值并触发失控事件提示或效果。
- 波次可持续推进（仅影响刷怪频率与强度）。
- 奖励由经验球触发，非清波触发。

---

## 6. 完成门禁记录模板（固定字段顺序）

- 成功观测结果（1 句）
- 关键证据（1-2 条；命令/日志片段）
- 未通过原因（仅当未通过）
- 待补验证（仅当未通过）

---

## 7. 开放问题（进入实现前需定稿）

- 过载资源最终采用冷却制、充能制还是混合制。
- 首版调性偏爆发清屏还是偏控场生存。
- 失控事件保护间隔是否启用，以及阈值跨越防抖策略。
