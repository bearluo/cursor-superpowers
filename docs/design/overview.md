# 赛博协议：过载与腐化（Roguelike 割草）总览

## 1. 项目定位

### 一句话卖点
玩家是觉醒实验体，能将异常能量编译为战斗协议；通过协议组合触发反应链清屏，并在腐化失控边缘主动过载，完成高风险高回报的战斗押注。

### 类型与节奏
- 类型：局外养成 + 局内双摇杆割草战斗（Roguelike/副本）。
- 单局目标时长：10-15 分钟。
- 核心节奏：每 2-3 分钟形成一次可感知的强度跃迁（新反应、更大范围或更强连锁）。

### 目标体验关键词
- 爽点密度高：短时间内持续获得构筑反馈。
- 反应可读：触发结果能看懂、能复现、可用于决策。
- 风险可预测：腐化阈值和失控代价明确，可主动押注而非被动受罚。

## 2. 核心体验闭环

### 局内（单局）
进入异常区 -> 清怪并获得经验/模块 -> 选择协议强化 -> 触发反应链 -> 主动过载放大窗口 -> 跨越腐化阈值并应对失控事件 -> 击败守门者 -> 深入或撤离结算。

### 局外（长线）
结算获得样本/零件/研究点 -> 提升稳定度与抑制器 -> 解锁协议研究分支 -> 改造机体插槽（手感向）-> 再次进入异常区。

## 3. 核心机制总览（MVP）

### 3.1 协议与 Stack 模型
MVP 使用 4 种协议：热（Ignite）、冷（Freeze）、毒（Toxin）、电（Arc）。  
统一规则：协议为敌人施加对应 Stack（0-100），当满足阈值/条件时触发反应事件。

### 3.2 反应表（最小可玩）
- 热 + 电：过热放电（小范围爆炸，连锁跳数 +1）
- 冷 + 电：超导（大范围减速/易伤，偏控场）
- 冷 + 热：热震（清空双 Stack 的高额瞬时爆发）
- 毒 + 热：燃毒（DOT 转扩散形态，偏清屏）
- 毒 + 电：神经脉冲（连锁附带短麻痹，偏生存）

反应可读性要求：每次反应触发至少提供下列反馈中的两项：图标/颜色、范围提示、数值或状态浮字。

### 3.3 腐化与阈值
腐化随时间推进、受击/濒死、高阶反应和过载行为上升；通过净化掉落或防御向模块缓解。  
MVP 采用三段阈值（建议 33/66/100），每段阈值都要改变战场风险预期。

### 3.4 主动过载
- 启动方式：主动键触发（冷却或电容消耗）。
- 持续窗口：短持续（建议 6-10 秒）。
- 收益：降低反应触发门槛或提升倍率，放大连锁与范围。
- 代价：腐化即时激增，窗口内增长加速，结束后进入短暂衰减期。

### 3.5 失控阈值事件
当跨越阈值或过载结束时腐化过高，触发失控事件：
- 追猎者降临（高风险高回报的精英压力）
- 协议反噬（短时封锁一个协议）
- 环境异常（电弧/毒雾/寒潮区域危害）

设计原则：失控事件不是纯惩罚，必须同时提供资源或构筑回报入口。

## 4. MVP 范围与边界

### 必须交付（本阶段）
- 玩法系统：4 协议 + 5 反应 + Stack 规则 + 反馈。
- 风险系统：腐化条、阈值、过载窗口、衰减、3 类失控事件。
- 关卡骨架：分区波次制、区末三选一奖励、至少 2 个守门者。
- 局外最小循环：稳定度/抑制器、协议研究、机体插槽。
- UI 与引导：协议列表、反应提示、腐化阈值预告、过载状态、首局基础教程。

### 暂不纳入（后续扩展）
- 协议扩展到 6-8 种及大规模反应树。
- 高腐化形态化异化系统。
- 重叙事分支与阵营结局深化。

## 6. 术语速览

- 协议：可施加状态并参与反应的基础战斗元素。
- Stack：协议状态层数，用于判断反应触发条件。
- 反应：两个或多个协议在条件满足时生成的特殊效果。
- 腐化：局内风险值，随行为增长并驱动阈值事件。
- 过载：主动开启的高收益高代价强化窗口。
- 失控事件：由高腐化触发的战场突发机制。
- 守门者：分区阶段性首领，检验构筑和操作。
